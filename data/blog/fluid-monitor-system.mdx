## 项目概述

本项目是在**江西宇瑞数字科技有限公司**实习期间主导开发的工业级管道流体实时监控系统，基于 **ESP32-P4 双核 400MHz** 主控，集成 7 英寸 LVGL 触摸屏 HMI、工业 ADC 精密采样、RS485 Modbus 通信以及四模式自动控制逻辑，已成功部署至 3 个实际工厂并稳定运行超过 30 天。

## 核心技术：ADC 多采样中值滤波

工业现场存在严重的**电磁干扰**（变频器、电机启停等），普通 ADC 单次采样误差极大。我实现了一套 **16 采样中值滤波算法**，将干扰峰值有效剔除：

```cpp
// esp32-p4_ADC.cpp - 核心 ADC 采样与校准
#include "esp_adc/adc_oneshot.h"
#include "esp_adc/adc_cali.h"
#include "esp_adc/adc_cali_scheme.h"

#define ADC_GPIO     GPIO_NUM_49    // 模拟输入（GPIO49 -> ADC2_CH0）
#define ADC_CHAN     ADC_CHANNEL_0

// ADC oneshot 单元初始化
adc_oneshot_unit_init_cfg_t init_config = {};
init_config.unit_id = ADC_UNIT_2;
adc_oneshot_unit_handle_t adc2_handle = NULL;
ESP_ERROR_CHECK(adc_oneshot_new_unit(&init_config, &adc2_handle));

// 通道配置（12dB 衰减，12bit 分辨率，覆盖 0~3.3V）
adc_oneshot_chan_cfg_t chan_cfg = {};
chan_cfg.atten = ADC_ATTEN_DB_12;
chan_cfg.bitwidth = ADC_BITWIDTH_12;
ESP_ERROR_CHECK(adc_oneshot_config_channel(adc2_handle, ADC_CHAN, &chan_cfg));

// Curve Fitting 校准（比线性校准精度更高）
adc_cali_curve_fitting_config_t cali_cfg = {};
cali_cfg.unit_id = ADC_UNIT_2;
cali_cfg.chan = ADC_CHAN;
cali_cfg.atten = ADC_ATTEN_DB_12;
cali_cfg.bitwidth = ADC_BITWIDTH_12;
adc_cali_create_scheme_curve_fitting(&cali_cfg, &cali_handle);
```

**中值滤波核心逻辑**（伪代码）：

```c
// 连续采样 16 次，排序后取中间值
int samples[16];
for (int i = 0; i < 16; i++) {
    adc_oneshot_read(adc2_handle, ADC_CHAN, &samples[i]);
    vTaskDelay(1 / portTICK_PERIOD_MS);
}
bubble_sort(samples, 16);
int filtered = samples[7];  // 中值

// 转换为电压再映射到流量
adc_cali_raw_to_voltage(cali_handle, filtered, &voltage_mv);
float flow = map_voltage_to_flow(voltage_mv);  // 4-20mA → 流量值
```

> **效果**：在 3 个工厂现场（含高压配电柜旁）测试，误差稳定在 **±1%** 以内。

## 四模式控制逻辑

系统支持 4 种独立运行模式，通过 LVGL 触摸屏切换：

```
┌────────────┐    用户切换    ┌────────────┐
│  手动模式  │◄──────────────►│  自动模式  │
│  直接控制  │               │  PID + 阈值│
└─────┬──────┘               └─────┬──────┘
      │                            │
┌─────▼──────┐               ┌─────▼──────┐
│  定时模式  │               │  快捷模式  │
│  时间表执行 │               │  预设场景  │
└────────────┘               └────────────┘
```

| 模式     | 触发条件           | 控制逻辑                                 |
| -------- | ------------------ | ---------------------------------------- |
| **手动** | 操作员主动触摸控制 | 直接继电器开关，即时响应                 |
| **自动** | 液位传感器阈值联动 | 液位低于下限 → 自动开泵；高于上限 → 关泵 |
| **定时** | 用户预设时间表     | RTC 时钟触发，支持周期性执行             |
| **快捷** | 预设场景一键调用   | 组合动作，适合日常操作                   |

## LVGL HMI 开发

7 英寸触摸屏（800×480，MIPI DSI 接口）的 HMI 界面由 **LVGL v8 + ESP_Brookesia** 驱动，实现了：

- **实时流量仪表盘**：圆形 Gauge 控件显示实时流量，动态更新
- **趋势折线图**：滚动显示历史流量曲线（最近 60 秒）
- **告警弹窗**：液位异常时全屏红色告警遮罩 + 蜂鸣器联动
- **参数配置页**：通过触摸输入校准参数、阈值设置

![HMI 触摸屏界面](/static/images/fluid-monitor/hmi-screen.jpg)

```c
// 主界面初始化（伪代码）
lv_obj_t *gauge = lv_meter_create(lv_scr_act());
lv_meter_set_scale_range(gauge, 0, 1000, 280, 240);  // 0~1000 L/h

// 告警状态更新（在 LVGL 任务中调用）
void update_alarm_ui(float flow_value) {
    if (flow_value > ALARM_HIGH_THRESHOLD) {
        lv_obj_set_style_bg_color(alarm_panel, lv_color_hex(0xFF0000), 0);
        lv_label_set_text(alarm_label, "⚠ 流量超限！");
        gpio_set_level(BUZZER_GPIO, 1);  // 蜂鸣器响
    }
}
```

## 性能优化

### 问题：CPU 占用过高（60%+）

**根本原因**：ADC 采样任务与 LVGL 渲染任务在同一核心上竞争，导致界面卡顿。

**解决方案**：

1. **任务核心分配**：将 ADC 采样 + 控制逻辑 pin 到 Core 1，LVGL 渲染 pin 到 Core 0
2. **DMA 数据传输**：屏幕刷新改用 DMA 传输，释放 CPU 资源
3. **采样间隔调整**：ADC 多采样从每 10ms 一次调整为 DMA 触发，CPU 空闲率提升

```c
// 任务创建时指定核心
xTaskCreatePinnedToCore(
    adc_sampling_task,   // 采样任务
    "adc_task",
    4096,
    NULL,
    5,                   // 优先级
    &adc_task_handle,
    1                    // 固定在 Core 1
);
```

**优化结果**：CPU 占用从 **65%** 降至 **28%**，HMI 帧率稳定在 **30fps**。

## 遇到的坑与解决方案

### 坑 1：ADC 读取值漂移严重

**现象**：同一流量下，连续读取的 ADC 值最大差异达到 ±200（12bit 满量程 4095），折算误差超过 5%。

**根本原因**：工厂变频器启停产生高频电磁干扰，直接耦合到信号线。

**解决方案**：

- 信号线改用屏蔽双绞线，屏蔽层单端接地
- ADC 采样从单次改为 **16次多采样中值滤波**
- 在 ADC 输入端增加 RC 低通滤波（10kΩ + 100nF，截止频率约 1.6kHz）

### 坑 2：校准方案选择错误

**现象**：使用 `adc_cali_line_fitting` 时，在 0.1V 以下和 3.2V 以上精度严重恶化。

**解决方案**：改用 `adc_cali_curve_fitting`，该方案通过多段曲线拟合补偿 ADC 非线性，全量程精度均匀。

```c
// ❌ 线性校准（低精度端点误差大）
adc_cali_line_fitting_config_t cali_cfg;

// ✅ Curve Fitting 校准（全量程均匀精度）
adc_cali_curve_fitting_config_t cali_cfg = {};
cali_cfg.unit_id = ADC_UNIT_2;
cali_cfg.atten = ADC_ATTEN_DB_12;
adc_cali_create_scheme_curve_fitting(&cali_cfg, &cali_handle);
```

### 坑 3：LVGL 任务与 ADC 任务死锁

**现象**：系统随机重启，错误日志显示 `Task watchdog got triggered`。

**根本原因**：LVGL 持有全局互斥锁时，ADC 任务尝试更新 UI 数据产生死锁，触发看门狗。

**解决方案**：引入消息队列解耦，ADC 任务只写队列，LVGL 任务在自己的时间片内读队列更新 UI。

```c
// 用队列解耦 ADC → LVGL 数据流
QueueHandle_t flow_data_queue = xQueueCreate(10, sizeof(float));

// ADC 任务：写队列（不直接操作 LVGL）
xQueueSend(flow_data_queue, &flow_value, 0);

// LVGL 任务：读队列更新 UI
float received_flow;
if (xQueueReceive(flow_data_queue, &received_flow, 0) == pdTRUE) {
    lv_meter_set_indicator_value(gauge, indicator, received_flow);
}
```

## 总结

这个项目是我第一次独立完成从**硬件选型、电路设计、固件开发到现场部署调试**的完整嵌入式工程项目。在这个过程中，我深刻体会到：

1. **工业级产品与 Demo 的最大差距是可靠性**：在实验室能跑通≠在工厂能稳定工作，电磁干扰、温度变化、现场操作规范都是必须考虑的因素
2. **多任务设计要从一开始就规划好**：任务核心分配、共享资源保护、消息传递机制，后期重构成本极高
3. **ADC 在工业场景下必须上滤波**：哪怕是最简单的中值滤波也能把精度从 5% 提升到 1%
