## 项目概述

智能物流看板是一个基于**前后端分离架构**的企业级物流管理系统，由本人独立负责**全栈开发**，提供订单管理、部门管理、物流信息查询、预警监控等完整功能，帮助企业实现物流信息的集中管理和实时监控。

## 功能模块

### 1. 订单管理

- 订单列表展示，支持筛选和搜索
- 订单详情查看和物流轨迹展示
- 订单状态管理（待发货、运输中、已签收、已退回、运输异常）
- 订单数据导出（Excel 格式）
- **批量导入订单**（支持 Excel/CSV 格式，实测支持 2000+ 条同时导入）
- 手动/批量刷新物流信息
- 订单删除/恢复（软删除）

### 2. 部门管理

- 部门列表展示（华东/华南/华北/华西/华中/海外事业部）
- 部门信息查看
- 按部门统计订单数据

### 3. 物流代理

- 物流信息查询与同步
- 物流状态自动识别（7 种标准状态）
- 物流公司代码映射（支持 100+ 家快递公司）
- 异常件状态智能识别
- 自动重试机制

### 4. 定时任务

| 任务名称                | 执行时间   | 功能                       |
| ----------------------- | ---------- | -------------------------- |
| logistics-daily-refresh | 每天 06:00 | 刷新所有活跃订单的物流信息 |
| logistics-noon-refresh  | 每天 12:30 | 刷新所有活跃订单的物流信息 |

> 两个任务都使用**分布式锁**防止并发执行，确保系统稳定性。

### 5. 操作日志

- 完整的操作记录（导入/导出/删除/更新等）
- 历史数据滚动查看
- 实时更新（每 3 秒自动刷新）
- 分页查询（每次加载 50 条）

## 关键技术难题与解决方案

### 挑战：客户要求同时上传 2000+ 订单

**问题分析**：

- 同步逐条插入导致请求超时
- 数据库连接池耗尽
- 前端界面卡死，用户体验极差

**解决方案**：

| 措施             | 实现方式                                                           |
| ---------------- | ------------------------------------------------------------------ |
| **异步队列处理** | 使用 **BullMQ** 消息队列，将订单导入任务丢入队列，后端异步消费处理 |
| **批量插入优化** | 将 2000+ 订单分批处理，每批 100 条，减少数据库往返次数             |
| **服务器扩容**   | 提升 MySQL 连接池上限，增加 Redis 缓存容量                         |
| **进度反馈**     | 前端轮询任务进度，实时显示处理状态（已完成/总数）                  |
| **超时处理**     | 延长 HTTP 超时时间，防止大文件上传断开                             |

**最终效果**：2000+ 订单导入时间从超时失败优化到 **3 分钟内完成**，系统稳定运行。

## 生产环境部署

### 部署方案

```
┌──────────────────────────────────────────────┐
│              客户端浏览器                       │
└──────────────────┬───────────────────────────┘
                   │
                   ↓
┌──────────────────────────────────────────────┐
│              Nginx 反向代理                     │
│         (负载均衡 + 静态资源缓存)                │
│         (SSL 终止 + 跨域处理)                    │
└──────────────────┬───────────────────────────┘
                   │
         ┌────────┴────────┐
         ↓                 ↓
┌──────────────┐   ┌──────────────┐
│ 前端静态文件  │   │ 后端 API 服务 │
│ (dist 目录)  │   │ (NestJS)     │
└──────────────┘   └──────────────┘
                           │
                   ┌───────┴───────┐
                   ↓               ↓
            ┌──────────┐   ┌──────────┐
            │  MySQL   │   │  Redis   │
            └──────────┘   └──────────┘
```

### 部署步骤

1. **Docker 打包**

   ```bash
   # 前端构建
   npm run build

   # 后端 Docker 镜像构建
   cd logistics-backend
   docker build -t logistics-backend:latest .
   ```

2. **Nginx 配置**

   ```nginx
   server {
       listen 8080;
       server_name localhost;

       # 前端静态文件
       location / {
           root /path/to/dist;
           try_files $uri $uri/ /index.html;
       }

       # API 反向代理
       location /api {
           proxy_pass http://localhost:3001;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
       }
   }
   ```

3. **Windows 服务器部署**
   - 使用 Windows Server 运行 Nginx
   - Docker Desktop 运行后端容器
   - MySQL 和 Redis 可 Docker 化或本地安装
